Links Back to top

A link should be descriptive of the place it's taking the user. It should also reflect the title (if possible) of the destination page, so the user knows they've arrived at the right place. Avoid using text such as, 'click here', or anything that does not make sense when read out of context. Screen readers for instance, can read link text in isolation without the surrounding content.

You can hide further descriptive text in a way that can be found by screen readers using a class of .visuallyhidden. The link should however remain descriptive of the place it's taking the user.

Buttons Back to top

<button>Standard</button>
<button class="btn-primary">Primary</button>
<button class="btn-secondary">Secondary</button>
<button class="btn-primary btn-loading">Loading</button>
<button class="btn-primary btn-proceed">Proceed</button>
<button class="btn-primary btn-back">Back</button>

Note: margin was added below the buttons in the example; it is not shown in markup above for simplicity.

Links that are styled as buttons should ideally be an action eg 'Continue', 'Book' or 'Confirm'.

Use the <button> element within a form or for clickable elements that do not have a meaningful href eg when you want to trigger something with Javascript

Continue to booking 
<a href="/booking-form" class="btn-primary">Continue to booking</a>
<button class="btn-primary">Send your message</button>

Disabled buttons Back to top

Buttons can be marked as disabled using the disabled attribute (recommended) or by adding a class of disabled to the button element.

<button disabled>Disabled standard</button>
<button class="btn-primary" disabled>Disabled primary</button>
<button class="btn-secondary" disabled>Disabled secondary</button>
<button class="btn-tertiary" disabled>Disabled tertiary</button>

Note: margin was added below the buttons in the example; it is not shown in markup above for simplicity.
