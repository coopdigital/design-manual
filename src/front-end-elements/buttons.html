Links Back to top

A link should be descriptive of the place it's taking the user. It should also reflect the title (if possible) of the destination page, so the user knows they've arrived at the right place. Avoid using text such as, 'click here', or anything that does not make sense when read out of context. Screen readers for instance, can read link text in isolation without the surrounding content.

Buttons Back to top

Buttons are constructed using Avenir Next Medium at 20px (1em), a 3px border radius, 15px 37.5px padding ($quarter-spacing-unit $half-spacing-unit * 1.25) and a box-shadow. On hover and the background colour is lightened. On focus and active the buttons take a 4px blue outline.

Contrast ratio 3.4 (AA over 18px or bold)
Contrast ratio 8.8 (AAA) 
<button class="btn-primary">Primary</button>
<button class="btn-primary">Primary</button>

Note: margin was added below the buttons in the example; it is not shown in markup above for simplicity.

Use the green primary button when you have one, clear call to action on your page.

Contrast ratio 5.4 (AA - AAA over 18px) 
<button class="btn-secondary">Secondary</button>

The use and style of the secondry button has been the cause of some debate and requires some explanation. Within this candidate set we are currently recommending this white version for the following reasons. The grey secondary button we have tried previously has not tended to be recognised as a button by our users. Using a colour for the secondary button was something else we tried, but to achieve a contrast ratio we were happy with was difficult as when used together the primary and secondary buttons felt too similar in thier intent.

There are pros to the use of the white button. It shows up well on all backgrounds, it uses the link blue as it's text colour and it is an obvious where it falls within the hierarchy. We will only find out how it performs after it has been tested with users.

In this form example the hierachy between the primary, secondary and tertiary actions are clear.
