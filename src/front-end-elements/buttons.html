Links

A link should be descriptive of the place it's taking the user. It should also reflect the title (if possible) of the destination page, so the user knows they've arrived at the right place. Avoid using text such as, 'click here', or anything that does not make sense when read out of context. Screen readers for instance, can read link text in isolation without the surrounding content.

Example Descriptive links

Online Wills Service


<a href="https://www.co-oplegalservices.co.uk/making-a-will/">More information about Wills</a>

You can hide further descriptive text in a way that can be found by screen readers using a class of .visuallyhidden. The link should however remain descriptive of the place it's taking the user.

Example Hidden descriptive text

More information about Wills on the Co-op Legal website


<a href="https://www.co-oplegalservices.co.uk/making-a-will/">
    More information about Wills<span class="visuallyhidden"> on the Co-op Legal website</span>
</a>

Links to download documents must indicate the type of file and size. Links can also be used with icons as long as they are universally understood. Remember, often the best icon is a text label.

Example Links with icons

Download your Will PDF (5mb)

Tweet about this awesome Design Manual

More information about Wills

<a href="#" class="link-icon-left"><span aria-hidden="true" class="icon icon-twitter"></span> Tweet about this awesome Design Manual</a>
<a href="#" class="link-icon-right">More information about Wills <span aria-hidden="true" class="icon icon-forward-arrow"></span></a>

Buttons

Buttons are constructed using Avenir Next Medium at 20px (1em), a 3px border radius, 15px 37.5px padding and a box-shadow. On hover and the background colour is lightened. On focus and active the buttons take a 4px blue outline.

Example

Contrast ratio 3.4 (AA over 18px or bold)
Contrast ratio 8.8 (AAA) 
<button class="btn-primary">Primary</button>
<button class="btn-primary-dark">Button</button>

Use the green primary button when you have one, clear call to action on your page.

When you have more that one call to action on the page use the blue primary, and/or a secondary button.

Use the <button> element within a form or for clickable elements that do not have a meaningful href eg when you want to trigger something with Javascript

Disabled buttons

Example

<button disabled>Disabled standard</button>
<button class="btn-primary" disabled>Disabled primary</button>
<button class="btn-secondary" disabled>Disabled secondary</button>
<button class="btn-tertiary" disabled>Disabled tertiary</button>

Buttons can be marked as disabled using the disabled attribute (recommended) or by adding a class of disabled to the button element.
