Overview

We provide a basic set of components that can be used to quickly build up a prototype that you can test. We recommend researching your service and building components that suit your users needs. If components you build are used across many services and are improving user experience then we will work them back into the Design manual.

Use of ‘alpha’ and ‘beta’ banners

Don’t use the words ‘alpha’ or ‘beta’ - they’re not well understood by people outside of digital.

The banner:

  • should be no more than two lines of text on a mobile
  • must link to a feedback function on each page of the service
  • should not be dismissible

And should say something like:

‘This is a new way to [report/apply/register etc service]. Help improve it - tell us what you think’

‘This is a new [thing]. Help improve it - tell us what you think’

Example

This is a new version of the [Service]. Help improve it - tell us what you think.
