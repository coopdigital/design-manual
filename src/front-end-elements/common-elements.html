Lists Back to top

Unordered list

Bullets should always:

  • use a lead-in line
  • make sense running on from the lead-in line
  • use lower case at the start of the bullet
  • have only one sentence per bullet point - use commas, dashes or semicolons to expand on an item
  • have more than 2 items
  • appear within the text and not as the whole bullet

Bullets should not:

  • be paragraphs masquerading as a list for a visual effect
  • take a full stop after the last bullet point
  • include ‘or’, ‘and’ after the bullets

The meeting agenda will be:

  • an overview of the last month
  • a presentation
  • a special announcement
<h4>The meeting agenda will be:</h4>
<ul>
    <li>an overview of the last month</li>
    <li>a presentation</li>
    <li>a special announcement</li>
</ul>

Ordered list

Numbers should only for a logical sequence of things, like a series of tasks that must be done in a certain order. Apart from this, same rules for bullet lists apply. For example:

How to put your shoes on:

  1. put on socks
  2. put on shoes
  3. tie laces
<h4>How to put your shoes on:</h4>
<ol>
    <li>put on socks</li>
    <li>put on shoes</li>
    <li>tie laces</li>
</ol>

Tables Back to top

Tables should only be used for tabular data, and contain a <caption> element with a description of the table contents for assistive technologies. The <table> element should be wrapped in a <div> element with the class table-overflow-x to allow scrolling on smaller devices which may not be able to fit the table within the width of the viewport.

2 of the Rochdale Pioneers
First name Last name
William Cooper
Samuel Ashworth
First name Last name 

<div class="table-overflow-x">
    <table>
        <caption><span class="visuallyhidden">2 of the Rochdale Pioneers</span></caption>
        <thead>
            <tr>
                <th scope="col">First name</th>
                <th scope="col">Last name</th>
            </tr>
        </thead>
        <tbody>
            <tr>
                <td>William</td>
                <td>Cooper</td>
            </tr>
            <tr>
                <td>Samuel</td>
                <td>Ashworth</td>
            </tr>
        </tbody>
        <tfoot>
            <tr>
                <td>First name</td>
                <td>Last name</td>
            </tr>
        </tfoot>
    </table>
</div>
