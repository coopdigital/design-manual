Colours

Overview

We use colour to guide people and aid communication with them. We should not rely on colour in our services, or use it purely for decoration.

Brand colour

Our brand blue should only be used for the logo. It should never be used as text or as a background colour. This is becuase it has poor contrast and may be difficult for those with low vision to read on screen.

Stylised text used in logos is exempt colour contrast requirements.

Text colours

Text colours meet web accessibility standards that the WCAG 2.0 specify for text and background color combinations. The colour contrast we look for on body text is 4.5:1 (AAA) and 3.1 (AA) on large text (above 18px or 14px bold).

If you are unsure about any of the colours you are using below are some tools that can help you to check and test them.

  • Web Aim's Color Contrast Checker
  • La Verou's Colour Contrast Analyser
  • Color Oracle (free color blindness simulator)
  • Information on use of colour from the WCAG 2.0 specification

Ultimately, there is no substitute for testing with real people of all abilities to see if your users are able to use your service.

Links colours

$coop-link-blue passes colour contrast at 4.5.1. so it can be used whereever neccessary. It works well when used within blocks of copy. Remember, if using on a dark background (or image) contrast test for legibility. It is fine to use links in white, grey or black to suit your needs but always make links self-evident by making them underlined. They must be clearly identifiable and distinguishable from surrounding text, by use of a presentation or context cue.

The exception to this rule is where main navigational links are already contextual and will be assumed by the user to be links and therefore do not need to follow this rule, for example, within the header, footer, any tertiary navigation or buttons.

The mouse-over state must use the hand / pointer (cursor:pointer) to indicate that the link is clickable.
