Text fields Back to top

All form controls should be accompanied with a label element associated to the field using the for attribute.

textarea elements should have their rows attribute set to the desired height.

<div class="form-row">
    <label for="input-1">Text input</label>
    <input id="input-1" type="text">
</div>

<div class="form-row">
    <label for="input">Inline text input</label>
    <input class="input-inline" id="input" type="text" size="10">
</div>

<div class="form-row">
    <label for="textarea-1">Textarea</label>
    <textarea id="textarea-1" cols="30" rows="4"></textarea>
</div>

Checkboxes and radio buttons Back to top

Radio buttons

Sets of radio buttons must be wrapped in a fieldset for accessibility purposes. The legend element must carry the label for the set of buttons.

What kind of will would you like?

Fill in this form based on decisions you and the other person have agreed.

If you’re not sure, or need to speak to them first, leave the questions blank and we’ll go through them when we call.

<fieldset>
    <legend>What kind of will would you like?</legend>
    <label class="tap-check" for="mirror-wills">
        <input type="radio" value="mirror" name="will-types" id="mirror-wills">Mirror wills
    </label>
    <label class="tap-check" for="single-will">
        <input type="radio" value="single" name="will-types" id="single-will">Single will
    </label>
    <label class="tap-check" for="not-sure">
        <input type="radio" value="not sure" name="will-types" id="not-sure">You're not sure
    </label>
</fieldset>

Checkboxes

If checkboxes are grouped, the set must be wrapped in a fieldset element. The legend element then carries the title for the grouped checkboxes.

Do you own:

or

<fieldset>
    <legend>Do you own:</legend>
    <label class="block-label" for="client_owns_a_business">
        <input type="checkbox" value="1" name="client[owns_a_business]" id="client_owns_a_business">a business
    </label>
    <label class="block-label" for="client_owns_agricultural_property">
        <input type="checkbox" value="1" name="client[owns_agricultural_property]" id="client_owns_agricultural_property">agricultural property
    </label>
    <label class="block-label" for="client_owns_foreign_property">
        <input type="checkbox" value="1" name="client[owns_foreign_property]" id="client_owns_foreign_property">foreign property
    </label>
    <label class="block-label" for="client_owns_trademarks_or_patents">
        <input type="checkbox" value="1" name="client[owns_trademarks_or_patents]" id="client_owns_trademarks_or_patents">trademarks or patents
    </label>
    <p class="or">or</p>
    <label class="block-label" for="client_owns_none_of_the_above">
        <input type="checkbox" name="client_owns_none_of_the_above" id="client_owns_none_of_the_above" data-none-of-the-above="">You do not own any of these
    </label>  
</fieldset>

Tap tiles Back to top

We no longer recommend the use of tap tiles but if your project uses them, don't worry. Consider upgrading to use the examples above for radio buttons and checkboxes. Rather than hide the browser default radio and checkboxes our new pattern enahances them to make the more usable.

Answer the question. Yes or No? 
<fieldset class="form-row">
    <legend>Answer the question. Yes or No.</legend>

    <input type="radio" id="radio-one" name="radio-group-1" class="tap-input">
    <label for="radio-one">Yes</label>

    <input type="radio" id="radio-two" name="radio-group-1" class="tap-input">
    <label for="radio-two">No</label>
</fieldset>

Large inputs Back to top

Large inputs can be created by simply increasing the font-size on the container element.

<div style="font-size: 1.5em;">
    <div class="form-row">
        <label for="large-1">Input</label>
        <input id="large-1" type="text">
    </div>
    <div class="form-row">
        <label for="large-2">Select</label>
        <select id="large-2">
            <option value="">Option 1</option>
            <option value="">Option 2</option>
        </select>
    </div>
</div>
