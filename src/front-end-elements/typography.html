Range and measure Back to top

We always range our text to the left (ragged right) in dark grey (#282828). Text, including headings and titles, should be sentence case.

The measure is the number of characters in a single line of a column of text.

“Anything from 45 to 75 characters is widely regarded as a satisfactory length of line for a single-column page set in a serifed text face in a text size. The 66-character line (counting both letters and spaces) is widely regarded as ideal. For multiple column work, a better average is 40 to 50 characters”. The Elements of Typographic Style Applied to the Web - 2.1.2 Choose a comfortable measure

To accommodate these guidelines we recommend that text in the recommended scale run over no more than 8 columns (on desktop). When working with columns run over no more than 4/5.

In this guide we have generally used 7 columns as it is more comfortable for reading on screen as eye movement is kept to a minimum. A good tool to help you test your measure is the Google Chrome extension 45 to 75.

Vertical rhythm Back to top

“Just as regular use of time provides rhythm in music, so regular use of space provides rhythm in typography, and without rhythm the listener, or the reader, becomes disorientated and lost”. Richard Rutter - Compose to a Vertical Rhythm

Our base typographic scale works in the same way. All of our spacing is derived from multiples of 8px. That is known as our baseline grid. This baseline allows us to space our text, images or any other page elements using these values.

“Space in typography is like time in music. It is infinitely divisible, but a few proportional intervals can be much more useful than a limitless choice of arbitrary quantities”. Robert Bringhurst

Our base font size 20px is set on a 32px 'leading' (the height of the line that the text sits on) - 8x4 = 32.

By using this system designers and developers get this for free. By overriding the values in our scale varibles in the Toolkit in your custom stylesheet you can create your own scale. However, you must have a good reason to do this and compose to your own well thought out scale.

Using titles Back to top

Titles should:

  • be sentence case
  • be less than 65 characters - Google cuts them off after this
  • be unique, clear and descriptive, no puns or single topic words
  • be front-loaded with key words if possible - people tend to scan the first couple of words only
  • use words you know your users actually search for
  • be active where possible, ‘Apply for…’, ‘Report…’, ‘Find…’
  • not have a full stop at the end
  • not use acronyms unless they are well-known, like EU, BBC

Orphans Back to top

One word, if left on the last line of a title or paragraph, is known as an 'orphan'. We recommend not having orphaned words in your service. It can be fixed by adding a non-breaking space (&nbsp;) between the final 2 words.

Typographic elements Back to top

HTML entities are small sequences of characters that reference a charcter in the Universal character set/unicode your web page uses. These entities can assist us in using typography correctly.

It is important to use the correct element to convey meaning both in your typography and your HTML. For example using <strong> is semantic — it describes the text it surrounds (eg. this text should be stronger than the rest of the text you've displayed) as opposed to describing how the text it surrounds should be displayed (this text should be visually bold).

In this case <strong> indicates how something should be understood and it will be read differently by assitive technologies such as screen readers.

Font style can be regular, bold or in some cases Medium.

Using standard HTML elements: strong, bold, deleted.

Abbreviation: HTML. Text links: link.

Typographic entities Back to top

There are certain entities that are useful to ensure that your web type is displayed correctly.

When you type " on your computer it is not a true quotation mark. True quotes are called smart quotes and the right one must be used according to whether you are opening (&ldquo;) or closing (&rdquo;) a quotation.

Hyphens (the character you get when you type - ) is a punctuation mark and should be used to hyphenate words in justified type. It can also be used in compound modifiers such as “fine-tuned”.

En dashes (&ndash;) should be used for connecting things that are related to each other by distance such as June–July

Em dashes (&mdash;) can be used in pairs like parentheses—that is, to enclose a word, or a phrase — or they can be used alone to detach one end of a sentence from the main body.

Scale variables Back to top

The typographic scale can be customised by overriding the default variable. To do so, set a new $typographic-scale variable with at least a base property containing all the default sizes. You can also pass optional extra breakpoints containing the sizes you want to modify as the screens get bigger.

The typographic scale and vertical rhythm of all elements across all devices can be controlled using this variable. This means that by setting a scale and $base-spacing-unit you can maintain vertical rhythm by using our helper mixin. For instance by using @include spacing('margin', 'bottom', 'base') will set a margin-bottom to the element. 'Base', refers to the $base-spacing-unit set in the scale variable. At each break point the margin-bottom will be changed to reflect the value set. In this case the base (mobile) margin bottom will be 32px and above the Medium break point will be 64px.

The mixin also allows you to pass other values that also fit to the scale (as they are derived by dividing the $base-spacing-unit by half each time). Therefore @include spacing('margin', 'bottom', 'quarter') would produce a margin-bottom: 16px above the medium breakpoint (in this example) and margin-bottom: 8px on mobile (or base break point).

$typographic-scale: (
  base: (
    body: 16px,
    h-mega: 46px,
    h1: 30px,
    h2: 20px,
    h3: 18px,
    h4: 18px,
    h5: 18px,
    h6: 13px,
    blockquote: 18px,
    lead: 20px,
    small: 13px,
    base-line-height: 1.7,
    base-spacing-unit: 32px,
  ),
  medium: (
    body: 20px,
    h-mega: 52px,
    h1: 46px,
    h2: 26px,
    h3: 22px,
    h4: 22px,
    h5: 22px,
    h6: 16px,
    blockquote: 22px,
    lead: 26px,
    small: 13px,
    base-line-height: 1.6,
    base-spacing-unit: 64px,
  )
) !default;
