Scale of sizes

It is important to have a typographic scale to create clear hierarchy and coherence for readers. By default we have a scale for larger and smaller devices. It is simple to add more breakpoints and sizes in the code to suit your user needs.

“Just as regular use of time provides rhythm in music, so regular use of space provides rhythm in typography, and without rhythm the listener, or the reader, becomes disorientated and lost”.

Richard Rutter - Compose to a Vertical Rhythm

The font we use is Avenir Next in demi bold, regular and medium. All of our spacing between lines, paragraphs and headings is derived from multiples of 8px. That is known as the baseline grid. This baseline allows us to space our text, images or any other page elements using regular values.

Horizontal space

The measure is the number of characters in a single line of a column of text.

“Anything from 45 to 75 characters is widely regarded as a satisfactory length of line for a single-column page set in a serifed text face in a text size. The 66-character line (counting both letters and spaces) is widely regarded as ideal. For multiple column work, a better average is 40 to 50 characters”.

The Elements of Typographic Style Applied to the Web - 2.1.2 Choose a comfortable measure

Text in the recommended scale run over no more than 7 columns of our grid (on desktop). When working with columns run over no more than 4/5. It is more comfortable for people reading on screen as eye movement is kept to a minimum. A good tool to help you test your measure is the Google Chrome extension 45 to 75.

Vertical space

All of our spacing is derived from multiples of 8px. That is known as our baseline grid. This baseline allows us to space our text, images or any other page elements using these values.

“Just as regular use of time provides rhythm in music, so regular use of space provides rhythm in typography, and without rhythm the listener, or the reader, becomes disorientated and lost”.

Richard Rutter - Compose to a Vertical Rhythm

Our base font size 20px is set on a 32px 'leading' (the height of the line that the text sits on) - 8x4 = 32.

By using this system designers and developers get this for free. By overriding the values in our scale varibles in the Toolkit in your custom stylesheet you can create your own scale. However, you must have a good reason to do this and compose to your own well thought out scale.

Type and colour

Use dark grey for text (#282828). Use other colours for text sparingly and always check colours have a high enough contrast.

Titles

Titles should:

  • be sentence case
  • be less than 65 characters - Google cuts them off after this
  • be unique, clear and descriptive, no puns or single topic words
  • be front-loaded with key words if possible - people tend to scan the first couple of words only
  • use words you know your users actually search for
  • be active where possible, ‘Apply for…’, ‘Report…’, ‘Find…’
  • not have a full stop at the end
  • not use acronyms unless they are well-known, like EU, BBC

Orphaned words

One word, if left on the last line of a title or paragraph, is known as an 'orphan'. We recommend not having orphaned words in your service. It can be fixed by adding a non-breaking space (&nbsp;) between the final 2 words.

Typographic elements

Unordered list

Bullets should always:

  • use a lead-in line
  • make sense running on from the lead-in line
  • use lower case at the start of the bullet
  • have only one sentence per bullet point - use commas, dashes or semicolons to expand on an item
  • have more than 2 items
  • appear within the text and not as the whole bullet

Bullets should not:

  • be paragraphs masquerading as a list for a visual effect
  • take a full stop after the last bullet point
  • include ‘or’, ‘and’ after the bullets

Example Unordered list

The meeting agenda will be:

  • an overview of the last month
  • a presentation
  • a special announcement
<h4>The meeting agenda will be:</h4>
<ul>
    <li>an overview of the last month</li>
    <li>a presentation</li>
    <li>a special announcement</li>
</ul>

Ordered list

Numbers should only for a logical sequence of things, like a series of tasks that must be done in a certain order. Apart from this, same rules for bullet lists apply. For example:

Example Ordered list

How to put your shoes on:

  1. put on socks
  2. put on shoes
  3. tie laces
<h4>How to put your shoes on:</h4>
<ol>
    <li>put on socks</li>
    <li>put on shoes</li>
    <li>tie laces</li>
</ol>

Always:

  • Use bold sparing
  • Avoid using italics (problematic for people with dyslexia)

It is important to use the correct element to convey meaning both in your typography and your HTML. For example using <strong> is semantic — it describes the text it surrounds (eg. this text should be stronger than the rest of the text you've displayed) as opposed to describing how the text it surrounds should be displayed (this text should be visually bold). In this case <strong> indicates how something should be understood and it will be read differently by assitive technologies such as screen readers.

Typographic HTML entities

HTML entities are small sequences of characters that reference a charcter in the Universal character set/unicode your web page uses. These entities can assist us in using typography correctly. There are certain entities that are useful to ensure that your web type is displayed correctly.

When you type " on your computer it is not a true quotation mark. True quotes are called smart quotes and the right one must be used according to whether you are opening (&ldquo;) or closing (&rdquo;) a quotation.

Hyphens (the character you get when you type - ) is a punctuation mark and should be used to hyphenate words in justified type. It can also be used in compound modifiers such as “fine-tuned”.

En dashes (&ndash;) should be used for connecting things that are related to each other by distance such as June–July

Em dashes (&mdash;) can be used in pairs like parentheses—that is, to enclose a word, or a phrase — or they can be used alone to detach one end of a sentence from the main body.

Front-end code

The typographic scale can be customised by overriding the default variable. To do so, set a new $typographic-scale variable with at least a base property containing all the default sizes. You can also pass optional extra breakpoints containing the sizes you want to modify as the screens get bigger.

The typographic scale and vertical rhythm of all elements across all devices can be controlled using this variable. This means that by setting a scale and $base-spacing-unit you can maintain vertical rhythm by using our helper mixin. For instance by using @include spacing('margin', 'bottom', 'base') will set a margin-bottom to the element. 'Base', refers to the $base-spacing-unit set in the scale variable. At each break point the margin-bottom will be changed to reflect the value set. In this case the base (mobile) margin bottom will be 32px and above the Medium break point will be 64px.

The mixin also allows you to pass other values that also fit to the scale (as they are derived by dividing the $base-spacing-unit by half each time). Therefore @include spacing('margin', 'bottom', 'quarter') would produce a margin-bottom: 16px above the medium breakpoint (in this example) and margin-bottom: 8px on mobile (or base break point).

More information can be found by looking at the toolkit code.

$typographic-scale: (
  base: (
    body: 16px,
    h-mega: 46px,
    h1: 30px,
    h2: 20px,
    h3: 18px,
    h4: 18px,
    h5: 18px,
    h6: 13px,
    blockquote: 18px,
    lead: 20px,
    small: 13px,
    base-line-height: 1.7,
    base-spacing-unit: 32px,
  ),
  medium: (
    body: 20px,
    h-mega: 52px,
    h1: 46px,
    h2: 26px,
    h3: 22px,
    h4: 22px,
    h5: 22px,
    h6: 16px,
    blockquote: 22px,
    lead: 26px,
    small: 13px,
    base-line-height: 1.6,
    base-spacing-unit: 64px,
  )
) !default;
