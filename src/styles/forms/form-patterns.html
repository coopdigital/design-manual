Labelling elements

Asking for name and title

Only ask for name and title if it’s essential.The format of people’s names can vary. Some do not fit into conventional UK formats. It can be unnecessary and uncomfortable for users to define themselves using a title, or a separate first name — surname format.

Never gather a title to assume gender.

Asking for gender and sex

Don’t ask.

As with gathering a ‘title’, some users may feel uneasy giving this information.

If you do need to know, ask about a person’s ‘gender’, not their sex. Allow them to tell us in their own words with a text entry field

If you’re gathering information to use it to address the user later, ask what pronoun they prefer - he/him, she/her, they/them.

But ultimately - if you do not need to know - do not ask.

Validation

We should be building services that are flexible enough to allow users to give data in a way that they would naturally. For example we should allow users to enter a postcode in upper or lower case, with or without a space.

Validation errors should only be triggered on mandatory fields. They must be factual and help the user move on.

Example

There's a problem

Check the form. You must:

  • Enter your full name
  • Enter an email address

Enter your full name

Enter an email address

Address

Only ask for a user’s postal address if it is essential information we need to fulfil the service. Telling users how you will use their address may affect their answer: their delivery address may be different to an address we need to verify their identity.

Progressive reveals

You might need to ask users for more information because of how they answered a previous question. You can use a ‘progressive reveal’ to do this so you only show the extra form field to the users who need to see it.

Only use progressive reveals to ask for one piece of information. If you need to ask for more than one thing, do so on a separate screen.

Example

What kind of will would you like?

Fill in this form based on decisions you and the other person have agreed.

If you’re not sure, or need to speak to them first, leave the questions blank and we’ll go through them when we call.

Multi-page forms

Only ask users for one type of information on screen at a time. This helps to keep them focused and makes the purpose of each page clear.

Continue and back

Allow users to move through the form with a ‘continue’ button and a ‘back’ link. Make the continue button the primary button on the page to encourage users to progress through the form.

Position your continue button on the bottom left of the page so it’s easier for users to see.

‘Continue’ buttons should always be active. If a user tries to progress without completing the necessary information, run validation messages to tell your user what the error is and how to fix it.

Progress trackers

A progress tracker gives your users an indication of how far through a form they are. But they can be more problematic than helpful — they can get in the way and confuse users.

Only use a progress tracker if you have thorough research to support the decision.
