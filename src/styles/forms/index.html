Forms

Designing good forms means making sure you're asking the right questions in the right way. Start by writing and structuring your questions thoughtfully.

Designing a good form

Before you start your form.

  • Know why you’re asking each question and what it’s needed for
  • Tell users why we’re asking
  • Use simple, direct language
  • Give information in context
  • Get information in context
  • Set expectations

Using form inputs

Any point that we ask users for data.

  • Form inputs
  • Drop down lists
  • Text entry fields
  • Text input
  • Text area
  • Submit buttons
  • Radio buttons and checkboxes

Form patterns

Common ways of asking users for data.

  • Asking for name and title
  • Asking for gender and sex
  • Validation
  • Address
  • Progressive reveals
  • Multi-page forms
  • Continue and back
  • Progress trackers
