Form inputs

Form inputs are any place in your form where you ask the user for data. They can include:

  • radio buttons and checkboxes
  • text entry fields (known as ‘inputs’ and ‘text areas’)
  • drop down menus
  • submit buttons

Radio buttons and checkboxes

Radio buttons and checkboxes allow users to select an answer from 2 or more options.

Use radio buttons when the user can choose only one answer from a list.

Use checkboxes when the user can choose multiple answers from a list.

If you’re using either radio buttons or checkboxes, you must:

  • style them in a way that makes it clear which option or options has been selected
  • allow users to click anywhere in the box
  • display the options a user can choose from top to bottom, not left to right — this makes it quicker for users to scan

Where possible, give users the option to indicate that none of the options apply to them. Some users need this to be an explicit option to know that they can move on.

Text entry fields

If you need your user to type information into your form, use a text entry field.

If you’re using text entry field, you must:

  • give them a label: ‘Full name’
  • allow users to click on the label to let them to start answer (clicking the label takes the cursor to the text entry field)
  • don’t use placeholder text (text that sits within the form field) — if a question needs extra clarification, use hint text instead

Types of text entry field

Depending on the answer you’re expecting, use either:

  • a one line entry box (known as a ‘text input’) for things like name, email and so on
  • multiple line entry box (known as ‘text area’) for things like feedback, additional information and so on

Text input

With a text input you must also make the length of the box appropriate for the amount of information needed: do not make it frustratingly short or intimidatingly long.

Text area

Use large text areas with caution. Many users find it hard to type, do not like to type or worry about articulating themselves. Large text areas can be daunting and discouraging.

If you have reason to use them, research them with mobile device users. Entering information using touch devices can be especially time-consuming.

Text areas should:

  • scale appropriately on mobile devices
  • be a size that allows the user to scroll outside of it
  • have ‘draggable’ corners so users can control the size of the text area
  • show a countdown of remaining characters (if you’re restricting the characters that can be typed in the text area)

Submit buttons

A submit button allow the user to send their data to us.

Submit buttons must:

  • be descriptive of what they’re doing, for example, ‘send order’
  • only appear at the end of your form
  • be the primary button on the page

If you’re designing a multi-page form, make the submit button look different to show users that they are at the end of the form.

Drop down lists

Drop down lists (also known as ‘select inputs’) can be hard to use because they:

  • hide content from the user
  • can slow users down
  • are less accessible than other options

There are alternatives to using a drop down list. Try:

  • checkboxes or radio buttons for short lists
  • type aheads (the user is offered a list of relevant suggestions as they type), such as Baymard's Country Selector, for longer lists such as country and county lists
  • text entry fields if you do not need the answer to be matched with a back-end system

Only use a drop down lists if you have thorough research to support your decision.

Our recommendations on drop down lists is based on research by the Nielsen Norman Group and Government Digital Service.
