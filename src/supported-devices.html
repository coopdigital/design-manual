Supported devices

Overview

We aim to provide a consistent experience across the various combinations of of browsers and devices that people use. However, we accept that, where there are no workarounds available, small variations are acceptable within 'Full' and, 'Partial' support.

You should use new technology and the latest features. But make sure you apply them in such a way that allows users who don’t have (or intentional disable) these features to access the same content.

Levels of support

Fully supported browsers

Our definition of Full support is that:

  • all content must be readable, usable and all functionality must work.
  • variations to presentation of content must be minimised.
  • where CSS layout is used, the CSS must be rendered by supported web browsers, so that a fully-styled version of the page is presented to the user.
  • pages should be developed to maximise the user experience for users of the web browser with the highest proportion of users

Partially supported browsers

Our definition of Partial support is that:

  • all core content must be readable and usable and the navigation must work
  • any degradation to functionality must be graceful degradation
  • any degradation to presentation must not obscure content

We don’t support IE 8, Windows Phone, Blackberry and Opera because not many of our customers use them.

Maintenance

We’ll update the browser support matrix as new versions become available and when our customers' choice of browsers change. It is fine to support devices and browsers not listed here if your analytics show that they are popular with your users.

* Browser Versions auto update without user intervention, so the Full supported version is the two most recent non-beta versions and Partial support is the third most recent non-beta version.
