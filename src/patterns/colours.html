Colours

We use colour to guide people and aid communication.

We do not:

  • rely on colour alone to convey meaning — consider people who are colour blind
  • use it just for decoration — apply it only when it’s helpful to the user
  • use blue text for anything other than links

We’re developing further guidance on using colour along with a secondary colour palette. We’ll update this page when it’s ready.

If you have a specific question you can contact Matt Tyas at matt.tyas@coopdigital.co.uk.

Brand colour

Our brand blue should only be used for the logo. Do not use it as text or as a background colour. It has poor contrast and can be difficult for those with low vision to read on screen.

Stylised text used in logos is exempt from colour contrast requirements.

Text colour

Our recommended text colours meet WCAG 2.0 web accessibility standards for text and background colour combinations. You should make sure that the colour contrast on body text is 4.5:1 (AAA) and 3.1 (AA) on large text.

If you are unsure about any of the colours you are using, check them using:

  • Web Aim's Color Contrast Checker
  • La Verou's Colour Contrast Analyser
  • Color Oracle (free color blindness simulator)
  • information on use of colour from the WCAG 2.0 specification

Ultimately, there’s no substitute for testing with real people, of all abilities.

Links colours

#00729a (link blue) passes colour contrast at 4.5.1. so it can be used whereever neccessary.

It works well within blocks of copy. But, if you’re using it on a dark background (or image), contrast test it for legibility.

You can use links in white, grey or black but underline them to make it evident that they’re links. They must be clearly identifiable and distinguishable from surrounding text — make it easy for users to scan.

However, you do not necessarily need to underline a link if it is:

  • a main navigational link, for example, www.co-operative.coop
  • within the header or footer

The mouse-over state must use the hand/pointer to indicate that the link is clickable.

Link text can be read out of context and should always tell the user where it’s taking them — no ‘click here’s’. Find out more about writing link text in the style guide.
