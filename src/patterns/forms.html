Element styles

Text inputs

Text inputs must use:

  • Avenir Next Regular
  • a 1px border in #6e6e6e changing to #00a1cc on focus
  • 12px 8px padding

The width of text inputs must be appropriate for the width of the information that the user will need to enter.

Text areas

Text areas must use:

  • Avenir Next Regular
  • a 1px border in #6e6e6e changing to #00a1cc on focus
  • 12px 8px padding

The height of text areas must be appropriate for the amount of the information that the user will need to enter. The re-size control on the bottom right must be draggable to allow for more content than the height allows.

Radio buttons

Wrap sets of radio buttons in a fieldset. The legend element must carry the label for the set of buttons. This allows screen readers users to associate the question with the answers.

Selects

Selects - also known as drop-down lists - should be avoided in most cases.

Selects can be a problem for users because they:

  • hide content from the user and can slow users down
  • can be harder for some users to manipulate
  • are less accessible than alternative form controls

Alternatives to using a Select:

  • check boxes work well for shorter lists, such as titles (Mr, Mrs etc.)
  • type ahead, such as Baymard's Country Selector, can help on longer lists such as country and county lists, and do the hard work to support common synonyms e.g Eire, Republic of Ireland; London, Greater London.
  • free text fields are often the best option when the content isn’t being used programmatically

Our recommendation on Selects is based on prior work done in this area by the Nielsen Norman Group and Government Digital Service.

Related: Redesigning the Country Selector - Labs - Baymard Institute.

Validation

We should let the user fill in form fields in a way that's most natural to them. So, for example, allow them to enter a postcode with or without a space.

When you do need to write validation messages, they should help the user fix the problem. They should be clear and unambiguous.
