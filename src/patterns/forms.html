Element styles

Text inputs

Text inputs must use:

  • Avenir Next Regular
  • a 1px border in #6e6e6e changing to #00a1cc on focus
  • 12px 8px padding

The width of text inputs must be appropriate for the width of the information that the user will need to enter.

Text areas

Text areas must use:

  • Avenir Next Regular
  • a 1px border in #6e6e6e changing to #00a1cc on focus
  • 12px 8px padding

The height of text areas must be appropriate for the amount of the information that the user will need to enter. The re-size control on the bottom right must be draggable to allow for more content than the height allows.

Radio buttons

Wrap sets of radio buttons in a fieldset. The legend element must carry the label for the set of buttons. This allows screen readers users to associate the question with the answers.

Example

Select menus

We do not recommend the use of select menus. If you feel you must use one please contact design@coopdigital.co.uk.

Select menus must use:

  • Avenir Next Regular
  • a 1px border in #6e6e6e changing to #00a1cc on focus
  • 12px 8px padding

Validation

We should let the user fill in form fields in a way that's most natural to them. So, for example, allow them to enter a postcode with or without a space.

When you do need to write validation messages, they should help the user fix the problem. They should be clear and unambiguous.

Find out how we write and structure validation messages.
