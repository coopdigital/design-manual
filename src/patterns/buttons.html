Links

A link should be descriptive of the place it's taking the user. It should also reflect the title (if possible) of the destination page, so the user knows they've arrived at the right place. Avoid using text such as, 'click here', or anything that does not make sense when read out of context. Screen readers for instance, can read link text in isolation without the surrounding content.

Example Descriptive links

Online Wills Service

Links to download documents must indicate the type of file and size. Links can also be used with icons as long as they are universally understood. Remember, often the best icon is a text label.

Buttons

Buttons are constructed using Avenir Next Medium at 20px (1em), a 3px border radius, 15px 37.5px padding and a box-shadow. On hover and the background colour is lightened. On focus and active the buttons take a 4px blue outline.

For desktop and tablet formats button width should be defined by the button content. Buttons should be ranged left in these formats; the left edge aligning with other content.
